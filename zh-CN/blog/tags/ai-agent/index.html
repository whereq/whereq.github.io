01 起源与哲学

图 1：两种截然不同的理念 — Hermes 押注认知深度，OpenClaw 押注连接广度。

OpenClaw 定义了个人 AI 智能体品类。最初名为 Clawdbot（2025 年 11 月），因商标问题先后改名为 Moltbot 和 OpenClaw。创始人 Peter Steinberger 于 2026 年 2 月加入 OpenAI，项目转由非营利基金会社区治理。

Hermes Agent 两个月后由 Nous Research 发布 — 该实验室是 Hermes、Nomos、Psyche 模型家族的创造者。OpenClaw 优化的是连接尽可能多的平台和工具，而 Hermes 优化的是深度：让智能体通过使用真正学习和进化。

Hermes 将记忆视为分层系统：始终在上下文中的事实、按需加载的程序性技能、可搜索的情景历史。OpenClaw 的记忆更简单但对团队隔离有效 — 每个助手有自己独立的存储，防止多代理设置中的交叉污染。

OpenClaw 的生态系统在绝对数量上远超 Hermes。ClawHub 有数千个社区技能，覆盖从 Gmail 到 Kubernetes 的各种场景。但这种规模带来了严重的安全代价 — ClawHavoc 供应链攻击和 1,467+ 确认的恶意技能代表严重的信任问题。Hermes 用广度换安全：更少的技能，但有预执行扫描和更安全的默认姿态。

15 迁移路径

OpenClaw → Hermes：内置迁移工具

hermes claw migrate 可导入你的人格（SOUL.md）、记忆、技能（自动转换为 agentskills.io 格式）、消息平台配置、API 密钥和 TTS 资源。支持交互式预览和 dry-run。技能冲突通过 skip/overwrite/rename 选项处理。可以在过渡期间同时运行两个代理。

🦞 Hermes → OpenClaw：手动过程

无内置迁移工具。需要手动映射记忆文件、技能格式转换和频道重新配置。自我进化技能系统在 OpenClaw 中没有等效物 — 你会失去复利自动化价值。预估工作量：12–21 小时。

16 最终结论

图 4：14 个对比维度的逐类别胜出汇总。

选择 Hermes 当你...

  • 希望智能体随时间真正变得更强
  • 优先考虑安全性（零 CVE，Docker 默认加固）
  • 需要数月交互的深度记忆
  • 想要 Serverless 部署，空闲近零成本
  • 使用中国 LLM 提供商（MiniMax、GLM、Kimi）
  • 是需要 RL 训练 / 轨迹生成的研究者
  • 想从 OpenClaw 迁移（内置迁移工具）

🦞 选择 OpenClaw 当你...

  • 需要最广泛的平台覆盖（25+）
  • 想要庞大的技能生态（ClawHub 13,700+）
  • 需要企业支持（NVIDIA NemoClaw）
  • 需要原生 Windows 支持
  • 想要配套应用（macOS、iOS、Android）
  • 需要多代理路由和按助手数据隔离
  • 已有成熟的 OpenClaw 部署
🤝 或者两个一起用

由于 Hermes 使用 Python 而 OpenClaw 使用 Node.js，它们在运行时不冲突。你可以将不同频道路由到不同代理，用 OpenClaw 覆盖广泛的消息平台，同时用 Hermes 处理受益于其学习循环的深度工作任务。hermes claw migrate 工具支持你准备好时的渐进式迁移。

01 Origins & Philosophy

Fig 1: Two fundamentally different bets on what an AI agent should prioritize.

OpenClaw defined the personal AI agent category. Originally called Clawdbot (Nov 2025), it was renamed Moltbot and then OpenClaw after trademark concerns. Creator Peter Steinberger joined OpenAI in February 2026, and the project moved to community governance under a non-profit foundation.

Hermes Agent arrived two months later from Nous Research — the lab behind the Hermes, Nomos, and Psyche model families. Where OpenClaw optimized for connecting to as many platforms and tools as possible, Hermes optimized for depth: making the agent genuinely learn and improve through use.

04 Memory Systems HERMES WINS

Hermes treats memory as a layered system: always-in-context facts, on-demand procedural skills, and searchable episodic history. OpenClaw's memory is simpler but effective for team isolation — each assistant has its own separate storage, preventing cross-contamination in multi-agent setups.

05 Skills Ecosystem OPENCLAW WINS

OpenClaw's ecosystem is far larger in raw numbers. ClawHub has thousands of community skills covering everything from Gmail to Kubernetes. But that scale came with severe security costs — the ClawHavoc supply chain attack and 1,467+ confirmed malicious skills represent a serious trust problem. Hermes trades breadth for safety: fewer skills, but with pre-execution scanning and a safer default posture.

06 Messaging Platform Support OPENCLAW WINS

Fig 4: OpenClaw covers ~25 platforms; Hermes covers 15+ with deeper integration per platform.

OpenClaw has LINE, Google Chat, Microsoft Teams, IRC, Twitch, Nostr, Zalo, and native iOS/Android/macOS companion apps — none of which Hermes currently supports. Hermes's advantage is cross-platform conversation continuity with semantic context restore, and deeper per-platform integration (Discord voice channels, Telegram webhook mode, WeCom WebSocket).

15 Migration Path

OpenClaw → Hermes: Built-in Migration Tool

hermes claw migrate imports your persona (SOUL.md), memory, skills (auto-converted to agentskills.io), messaging configs, API keys, and TTS assets. Interactive preview with dry-run support. Skill conflicts are handled with skip/overwrite/rename options. You can run both agents side by side during transition.

🦞 Hermes → OpenClaw: Manual Process

No built-in migration tool. Requires manual mapping of memory files, skill format conversion, and channel reconfiguration. The self-improving skills system has no OpenClaw equivalent — you lose compounding automation value. Estimated effort: 12–21 hours.

16 Final Verdict

Fig 5: Category-by-category winner summary across all 14 comparison dimensions.

Choose Hermes When You...

  • Want an agent that genuinely improves over time
  • Prioritize security (zero CVEs, Docker hardened defaults)
  • Need deep memory across months of interactions
  • Want serverless deployment with near-zero idle costs
  • Work with Chinese LLM providers (MiniMax, GLM, Kimi)
  • Are a researcher needing RL training / trajectory generation
  • Want to migrate from OpenClaw (built-in migration tool)

🦞 Choose OpenClaw When You...

  • Need the broadest possible platform coverage (25+)
  • Want a massive skills ecosystem (13,700+ on ClawHub)
  • Need enterprise support (NVIDIA NemoClaw)
  • Require native Windows support
  • Want companion apps (macOS, iOS, Android)
  • Need multi-agent routing with per-assistant isolation
  • Already have an established OpenClaw deployment
🤝 Or Run Both

Since Hermes uses Python and OpenClaw uses Node.js, they don't conflict at runtime. You can route different channels to different agents, use OpenClaw for broad messaging coverage while Hermes handles deep-work tasks that benefit from its learning loop. The hermes claw migrate tool supports gradual migration when you're ready.
